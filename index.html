Definición Literaria del Ambiente (Estructuras Heterogéneas)

Se modela la información utilizando la nomenclatura formal de la cátedra para representar las entidades en memoria RAM antes de cualquier proceso de persistencia [cite: TEMA 7_ Registros.md].

1. Estructura de Sub-registro (Campo Continente)
REGISTRO: Fecha
CAMPOS:
    dia  : Entero   // Rango válido: 1..31
    mes  : Entero   // Rango válido: 1..12
    anio : Entero   // Representación de cuatro dígitos
FIN_REGISTRO
2. Registro Principal (Entidad de Servicio)
REGISTRO: AlojamientoTech
CAMPOS:
    id_apartado    : Entero        // Campo Contenido - CLAVE PRIMARIA (Identificador unívoco)
    denominacion   : AN(50)        // Campo Contenido - Tipo Alfanumérico para descripción comercial
    capacidad_max  : Entero        // Campo Contenido - Capacidad física de la unidad
    tarifa_diaria  : Real          // Campo Contenido - Datos numéricos de precisión con punto flotante
    fecha_alta     : Fecha         // CAMPO CONTINENTE - Registro anidado para control temporal
    disponibilidad : Carácter      // Campo Contenido - Operaciones binarias lógicas ('D'= Disponible, 'O'= Ocupado)
FIN_REGISTRO
Justificación Científica del Modelo Estructurado
  • Propiedad de Heterogeneidad: Como dictamina la teoría, el registro AlojamientoTech unifica bajo una misma entidad lógica diferentes tipos elementales (Entero, Real, Carácter) y complejos (Fecha), optimizando el tratamiento de la información como un bloque de memoria indivisible [cite: TEMA 7_ Registros.md].
  • Campos Continentes vs Contenidos: Los campos como id_apartado o `tarifa_diaria` son de tipo contenido ya que almacenan un dato atómico simple [cite: TEMA 7_ Registros.md]. En contraposición, fecha_alta es un campo continente porque subdivide su estructura interna en múltiples tipos elementales independientes (día, mes, año) [cite: TEMA 7_ Registros.md].
  • Acceso Mediante Selector de Campo: Para leer o escribir en las jerarquías más profundas de la estructura en los algoritmos, se utiliza el operador de indirección directa (punto) [cite: TEMA 7_ Registros.md]. Por ejemplo, la validación temporal se computa mediante el selector:
    AlojamientoTech.fecha_alta.anio [cite: TEMA 7_ Registros.md]
  • Análisis y Elección de la Clave Primaria: El campo id_apartado fue determinado como la Clave Primaria debido a que cumple con el axioma de unicidad absoluta: no se repite bajo ninguna circunstancia entre distintos registros de la secuencia [cite: TEMA 7_ Registros.md]. Campos numéricos de tipo Real como la tarifa o alfanuméricos como la denominación se descartan por completo ya que admiten duplicidad de valores en el procesamiento masivo de datos, lo que rompería la integridad de la estructura [cite: TEMA 7_ Registros.md].
